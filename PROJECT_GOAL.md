# Project Goal

## Goal Identity

- Goal ID: `Q2_SCI_DELIVERY_MODE`
- Delivery target: continuously advance `ghy001122/PINN` until it contains an evidence-sufficient, logically closed, reproducible, academically ethical Q2 SCI manuscript draft, submission package, supplementary material, and reviewer-defense matrix.
- Acceptance boundary: this is a project-delivery target, not a guarantee of journal acceptance.

## North-Star Scientific Claim

In a phase-transition/memristor synthetic numerical digital twin, sparse electrical terminal observations are insufficient to uniquely recover arbitrary complete hidden fields. Identifiability audits, prior calibration, and target-space reduction can make effective thermal-dissipation recovery more stable under explicit boundaries; active protocols, multilayer devices, and multi-terminal observations then define how the recoverable region expands or fails.

## Mandatory Research Filter

Every task must directly support at least one of:

- the core claim;
- a main figure or table;
- a method equation;
- a baseline or ablation;
- generalization, robustness, or uncertainty evidence;
- reviewer defense;
- a publishable negative result;
- reproducibility or the submission package.

Do not execute tasks whose manuscript value cannot be stated. Select one active bottleneck per round using:

\[
\frac{\text{manuscript value}\times\text{probability of useful evidence}\times\text{reviewer-defense value}}
{\text{time}\times\text{compute}\times\text{risk}}.
\]

## Evidence Lifecycle

Every actual experiment follows:

```text
config -> implementation -> test -> JSON/CSV -> figure/table -> report -> claim matrix -> manuscript sentence
```

Smoke, finite output, loss decrease, a local Jacobian, a response surface, mini-STL, or Fourier integration is not an automatic positive claim.

Every round must close with:

- actual completed work;
- change in distance to the delivery goal;
- claim upgrades or downgrades;
- new blockers;
- the next single highest-priority task;
- a disposition: continue, stop, downgrade, or move to manuscript assembly.

Update the compact status chain and relevant registries without duplicating the same prose.

The detailed reusable workflow, run classes, gate card, and failure routing live in `docs/research_strategy/sci_delivery_pipeline.md`.

## Must-Have Definition Of Done

- Frozen GT and all mainline evidence reproduce from documented commands.
- The `gamma_sub` claim closes confounding, calibration, noise, off-grid, observation-count, protocol, and limitation loops.
- At least one provenance-backed external quantitative anchor is completed.
- Main figures, main tables, Methods equations, claim matrix, and manuscript sentences map one-to-one.
- A complete manuscript draft and supplementary material exist.
- Code/data provenance and explicit reproduction commands exist.
- A reviewer-defense matrix covers likely objections and negative evidence.
- Every positive manuscript claim is `supported` or `qualified_supported`.
- No unresolved contradiction exists between code, outputs, reports, registries, and manuscript wording.

## Upgrade Gates

- P1 strict gate: `E_T <= 0.25`, `E_m <= 0.25`, interface residual `<= 0.05`, multi-seed success rate `>= 0.70`.
- P2 thermal block: a full-rank protocol must pass a nonlinear noisy-inverse gate with predeclared error, success, and coverage thresholds.
- Generalization: formal within-family geometry, stack, interface, and pulse OOD results.

## Stretch Goals

- Dynamic multi-terminal 2D field inverse.
- Canonical Seiler-style multi-head STL.
- Conditional Fourier/F-SPS superiority.
- Compositional, material-routed neural operator.

Stretch failure cannot block paper delivery; it only limits claim level.

## Stop, Lock, Or Redirect Rules

- Gate passed: lock evidence and stop unproductive tuning.
- Budget exhausted without passing: mark `failed_but_informative` and write the boundary result.
- Physical equation, source, coordinate, or observation-model error found: stop extensions and repair the foundation.
- Non-identifiability found: reduce the parameter space, add observations, or downgrade the claim.
- Required real experiments are unavailable: use a provenance-backed public-data anchor or state the limitation.
- Never obtain success by changing GT, altering true labels, deleting failed seeds, hiding failed blocks, or changing thresholds after results are known.

## User Confirmation Boundary

Explicit user confirmation is required only for:

- modifying frozen GT;
- deleting or overwriting uncommitted user files;
- adding a major dependency;
- changing the core manuscript line;
- using external data with unclear provenance;
- exceeding a predeclared high-cost experiment budget;
- force-push, history rewrite, or another irreversible operation.

Within all other boundaries above, Codex should proceed autonomously.
