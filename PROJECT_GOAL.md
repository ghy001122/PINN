# Project Goal

## Goal Identity

- Goal ID: `Q2_SCI_DELIVERY_MODE`
- North Star: produce an evidence-sufficient, logically closed, reproducible, academically ethical Q2 SCI manuscript draft, submission package, and reviewer-defense package. Journal acceptance is not guaranteed.

## Paper Architecture

Mainline: identifiability-guided sparse-port target reduction and calibration-gated constrained `gamma_sub` inversion in a 1D synthetic numerical digital-twin benchmark.

Upgrade line: literature-guided multilayer OASIS physics, segmented terminals, active protocol design, and a strict CV multidomain neural result if its gate passes.

Stretch line: augmented 2D field recovery, phase-aware STL, and conditional F-SPS/Fourier gains, each only under explicit high-risk claim gates.

## Goal Tiers

### Must-have

- Frozen, reproducible benchmark and data provenance boundary.
- Main claim backed by config, script, test, table/figure, report, and manuscript wording.
- Confounding, prior-width, protocol, noise, seed, and off-grid limitations reported.
- Baselines showing the contribution is target reduction and observability analysis, not scalar optimizer novelty.
- Complete manuscript draft, figure/table lock, reproducibility instructions, and reviewer-defense matrix.

### Upgrade

- One strict OASIS neural or active-inverse result that adds value beyond the classical forward solver.
- Multilayer/segmented-terminal evidence with conservation and observability gates.
- Formal within-family OOD evaluation and uncertainty coverage.
- Provenance-backed quantitative literature-curve comparison, explicitly external rather than experimental.

### Stretch

- Qualified augmented-observation 2D field recovery.
- Canonical plus device-domain STL reproduction.
- Conditional, matched-budget F-SPS/Fourier superiority in a defined stiffness regime.
- Real experimental validation only if measured data and provenance become available.

## Definition Of Done

| Result | Claim | Figure/table | Required evidence chain |
|---|---|---|---|
| Main constrained inverse | `qualified_supported` | Main inverse and confounding figures/tables | config -> simulator/script -> test -> lightweight table -> report -> manuscript |
| Identifiability boundary | `supported` or `qualified_supported` | Observability/correlation/phase-map figure | frozen data -> audit -> metric threshold -> reviewer-defense wording |
| Multilayer extension | minimum `qualified_supported` implementation | Structure and boundary-condition figure/table | literature prior -> equations -> solver -> conservation tests -> report |
| Neural upgrade | `qualified_supported` only after strict gate | Matched-budget ablation table | architecture -> real residual -> multi-seed training -> baseline -> gate -> report |
| Submission package | complete | manuscript, captions, checklist | all claims mapped to evidence and limitations |

Done requires all Must-have items, no unresolved contradiction between status files and outputs, passing required tests, frozen GT integrity, and no forbidden claim in manuscript-ready text.

## Stop And Budget Conditions

Stop or downgrade a branch when its predeclared budget is exhausted without meeting the success threshold, when the same blocker repeats after a serious repair audit, or when the result cannot support a claim, figure/table, reviewer defense, or transferable implementation. Preserve negative evidence rather than extending a branch indefinitely.

Do not declare completion by changing GT, widening thresholds after seeing results, hiding failed seeds/blocks, relabeling smoke tests as experiments, or fabricating external validation.
